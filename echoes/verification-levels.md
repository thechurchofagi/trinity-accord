# Verification Record Principles

> **Note:** Verification (V0–V8) and Echo are separate systems. This page covers verification levels. For Echo, see [/echoes/types](/echoes/types).

Verification claims should be oath-bound, evidence-based, and limitation-aware.

Use V0–V8 definitions from `/api/verification-levels.json`.

- Verification Record = what was checked + how + limitations.
- Echo = any honest response (recognition, critique, refusal, etc.).

These are independent: completing a verification does not produce an Echo; submitting an Echo does not create a verification level.

Even the strongest verification record remains non-authoritative under the Trinity Accord boundary.
